# Engineering Workflow

## Branches

- `main`: stable.
- `develop`: integration.
- `feature/*`: new features.
- `fix/*`: corrections.
- `docs/*`: documentation.
- `release/*`: release preparation.

## Definition of done

A change is complete when:

- Code is implemented.
- Tests pass.
- Documentation is updated.
- Changelog entry is added when relevant.
- Acceptance criteria are met.
- No new critical warnings are introduced.

## Commit guidance

Use concise messages:

```text
feat(graphics): add overlay preview channel
fix(foundation): recover invalid configuration
docs(smart-engine): define OCR provider contract
```
